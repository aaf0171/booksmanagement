Feature: Send activation email after borrower creation

Description:
After a borrower and their login account are successfully created, the system must
send an activation email to the borrower's email address.

The email contains an activation link with the activation token so the borrower can
activate their account and access the application.

The account is created in a non-activated state by default. The borrower must validate
their email address via the activation link received by email.

-------------------------------------------------------------------------------

Business rules

- An activation email is sent only after successful creation of both the borrower
  and their login account.
- The email is sent to the address provided during borrower creation.
- The email contains an activation link with the activation token.
- The email must NOT be sent if borrower or login creation fails.
- The email sending logic is delegated to a dedicated service — the borrower
  creation service must NOT contain email composition logic.
- The email subject and content must be swappable without impacting business logic.
- The activation token used in the link is already generated by a separate service
  (see books-0014). This ticket only handles the email sending.

-------------------------------------------------------------------------------

Architecture

BorrowerService
       │
       ▼
ActivationEmailService
       │
       ▼
JavaMailSender

BorrowerService: orchestrates borrower creation and triggers the email.
ActivationEmailService: builds the activation message and delegates sending.
JavaMailSender: performs the SMTP delivery.

-------------------------------------------------------------------------------

Service contracts

BorrowerService

After persisting the borrower and login:

1. Retrieve the activation token for the newly created login.
2. Delegate to ActivationEmailService to send the activation email.
3. If email sending fails, log the error but do NOT roll back the borrower creation.

ActivationEmailService

Receives: borrower email address, activation token.

Responsibilities:
1. Build a MimeMessage with:
   - To: borrower's email address
   - Subject: configurable (e.g. "Activate your account")
   - Body: HTML or plain text containing the activation link
2. Send the message via JavaMailSender.
3. Return success/failure status.

The borrower service must NOT know the email subject, body, or link format.

-------------------------------------------------------------------------------

Technical constraints

- The application uses Spring Boot.
- Email sending relies on JavaMailSender (Spring's standard abstraction).
- The SMTP server must be fully configurable via Spring configuration
  (application.yml and profiles).
- No SMTP information (host, port, authentication, etc.) may be hardcoded.
- The solution must work with a development SMTP server (Mailpit) and a
  production SMTP server with TLS/SSL.

-------------------------------------------------------------------------------

Configuration

application.yml example (development — Mailpit)

spring:
  mail:
    host: localhost
    port: 1025
    username: ""
    password: ""
    properties:
      mail:
        smtp:
          auth: false
        transport:
          ssl:
            enable: false

application.yml example (production)

spring:
  mail:
    host: smtp.example.com
    port: 587
    username: noreply@example.com
    password: "${SMTP_PASSWORD}"
    properties:
      mail:
        smtp:
          auth: true
          starttls:
            enable: true
        transport:
          ssl:
            enable: false

-------------------------------------------------------------------------------

Out of scope

The following are NOT part of this ticket:

- Activation token generation (books-0014)
- Token persistence
- Token validation
- Token expiration
- Email retry / resend
- Asynchronous sending (RabbitMQ, Kafka, Spring @Async, etc.)
- HTML template customisation (Thymeleaf, etc.)

The objective is solely to put in place a clean architecture for sending an
activation email via a dedicated service, using JavaMailSender and externalised
SMTP configuration.

-------------------------------------------------------------------------------

Acceptance criteria

✓ Activation email is sent after successful borrower + login creation.
✓ Email is sent to the borrower's email address provided at creation.
✓ Email contains a clickable activation link with the activation token.
✓ Email is NOT sent if borrower or login creation fails.
✓ BorrowerService does NOT contain email composition logic.
✓ ActivationEmailService handles email building and sending.
✓ JavaMailSender is used for SMTP delivery.
✓ SMTP configuration is externalised in application.yml (no hardcoded credentials).
✓ Configuration works with Mailpit in development.
✓ Configuration works with a TLS-enabled SMTP server in production.
✓ Email subject and content can evolve without changing BorrowerService.

Example cURL

Create borrower (triggers activation email)

curl --request POST \

     --header 'Content-Type: application/json' \
     --data '{
       "firstname": "Alice",
       "lastname": "Dupont",
       "email": "alice.dupont@example.com",
       "username": "alicedupont"
     }'

Expected: borrower created with 201, activation email sent to alice.dupont@example.com.
